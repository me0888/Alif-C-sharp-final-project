# Управление рисками предприятия (ERM) приложение

Это приложение разработано для управления организационными рисками предприятия (ERM) согласно предоставленному техническому заданию.

## Примечание

Это приложение является учебным проектом курса в Алиф Академии.
Приложение не является исчерпывающим и не охватывает все аспекты управления рисками предприятия.

## Описание

Управление рисками предприятия (ERM) - это система управления организационными рисками, которая позволяет организациям эффективно управлять различными видами рисков, включая внутренние и внешние.

## Функциональности

1. **Создание профилей рисков:** Приложение позволяет создавать профили рисков для различных бизнес-процессов или подразделений, включая название риска, его описание, вероятность возникновения, влияние на бизнес и потенциальное решение.

2. **Управление сущностью "Риск":** Приложение обеспечивает управление сущностью "Риск" и его характеристиками, такими как тип, описание, вероятность и влияние на бизнес.

3. **Хранилище рисков:** Приложение обеспечивает динамическое хранилище для эффективного хранения и управления рисками, а также функционал добавления, удаления, поиска и фильтрации рисков.

4. **Миграция и асинхронное API:** Реализованы механизмы миграции с временного хранилища на постоянное файловое хранилище, а также миграция на асинхронное API для повышения производительности и доступности.

5. **ASP .NET Core веб-сервис:** Приложение использует ASP .NET Core и разработано как веб-сервис для взаимодействия с пользователями через HTTP-запросы, включая маршруты, контроллеры и авторизацию по API-KEY.

6. **Redis для кэширования:** Приложение использует хранение профилей риска в кэш-памяти на основе механизмов Redis.

## Технические детали

Проект разработан на языке программирования C# с использованием ASP.NET Core для создания Web API. В проекте реализованы многоуровневая архитектура, валидация данных, CRUD-операции для управления рисками, а также асинхронное взаимодействие с базой данных с использованием Entity Framework Core. Для хранения данных используется реляционная база данных PostgreSQL и Redis (для кэширования).

## Требования к окружению

1. **.NET Core SDK**

2. **СУБД (PostgreSQL)**

3. **Redis (для кэширования)**

## Установка и запуск

1. **Перейдите в директорию проекта:** 

	```
	cd src\Erm.PresentationLayer.WebApi
	```

2. **Установите зависимости:** 

	```
	dotnet restore
    dotnet tool restore
	```

3. **Настройте подключение:** Настройте подключение к PostgreSQL и Redis в файле appsettings.Development.json.

4. **Выполните миграции для создания базы данных:** 

	```
	dotnet ef database update -p ../Erm.DataAccess/Erm.DataAccess.csproj -s Erm.PresentationLayer.WebApi.csproj
	```
	
5. **Запустите приложение:** 

	```
	dotnet run
	```

## Использование API

API предоставляет ряд эндпоинтов для управления рисками. Документация API доступна после запуска приложения по адресу http://localhost:5000/swagger/index.html.
